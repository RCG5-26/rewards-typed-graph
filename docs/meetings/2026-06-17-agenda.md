### 4. Close the blocker gaps from the review (0:25–0:55, 30 min) — the real work
Each has a pre-filled recommendation in [`schema-prepdoc-meeting1.md`](schema-prepdoc-meeting1.md). Decide, assign, move on. These sit on the hero / Layer 4 / benchmark paths, so they are worth the time.

- **B1 — Staleness mechanism.** Graph-write-service chokepoint runs propagation in the same transaction; add a `user_balances` trigger as a backstop? (Alan, Raq signs the contract)
- **B2 — Dependency coverage.** MVP staleness = node-valued only (balance, status, bonus); edge-valued (earn rate, base ratio) explicitly deferred and documented. Agree? (Alan, Michael)
- **B3 — The multi-hop bridge.** `TransferPartner.lands_in_program_id → RewardProgram`, or unify partners into RewardProgram. The redemption hero path cannot traverse without this. (Alan, Michael)
- **B4 — UserBalance uniqueness.** Unique `(user_id, program_id)`; WalletAgent updates in place, version++. (Alan)
- **B5 — Polymorphic dependency ref.** Keep table-per-type; node-reference integrity is an app-level check + orphan sweep, not an FK. Michael builds the verifier knowing this. (Alan, Michael)

### 5. Integration contract between lanes (0:55–1:10, 15 min) — Raq
The seams that let four people build in parallel. Agree on the shape today:
- **One write path.** Every mutation goes through the graph-write service (validation + staleness + no-cycle). No agent writes around it.
- **Canonical schema artifact.** `schema/schema.sql` + shared types (`schema/types.ts` / Python), one source of truth, both stacks validate against it. Alan ships right after lock.
- **`DEPENDS_ON_STATE` shape + `node_type` strings frozen.** Orchestrator and redemption agent are blocked until these are final. Freeze them in this meeting.
- **Mutation-log event shape.** What Val's sidebar subscribes to. Lock the fields so frontend is not blocked on backend.
- **Graph-typed tool fragment envelope.** `{nodes, edges}` + provenance, upsert by id/slug + version (ties to review I3).
- **Seed fixture, stable IDs.** 20 cards, top 50 MCCs, Chase UR + Amex MR partners (Hyatt/United/ANA), demo persona. All lanes build against the same IDs.

### 6. Benchmark integrity + demo scope calls (1:10–1:20, 10 min) — Michael + Raq
- **I1 — Baselines are output sinks only.** The free-text baseline must not use `DEPENDS_ON_STATE`; it persists final Plan/PlanStep rows for scoring only. Protects the wins-by-kind result. (Michael)
- **I2 — `benchmark_query_id`** on Plan so the same query joins across architectures. (Michael)
- **I3 — Minimal `ExternalQuote` node** so tool results merge as graph fragments, not JSON. (Michael, Alan)
- **I4 — Business-class demo beat: in or out?** Recommend cut unless I3 is trivial; keep the balance-change re-plan as the single hero moment. (Raq)
- **Gold corpus as-of date.** Pick the date the ground-truth ratios are frozen at; it defines the hallucination metric. (Michael)
